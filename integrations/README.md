# Integrations

This directory documents system-level interactions between contexts and project repositories.

Use it for:

- synchronous APIs
- asynchronous events
- shared contracts
- critical data flows
- versioning rules for interfaces

Keep `SYSTEM-MAP.md` high-level. Put detailed interaction descriptions here.

Recommended per-integration sections:

- participants
- direction of dependency
- contract or message type
- target architecture
- current state / known divergences
- related ADRs
