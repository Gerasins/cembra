# Contexts

This directory contains one folder per domain context.

Recommended structure:

```text
contexts/
  <context-name>/
    README.md
```

Each context document should be lightweight and should cover:

- purpose
- target boundary
- current state / known divergences
- key terms specific to the context
- incoming integrations
- outgoing integrations
- related project repositories
- related ADRs

Create context folders only when there is real content to add.
