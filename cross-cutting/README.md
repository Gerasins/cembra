# Cross-Cutting Rules

This directory contains architecture rules that apply across multiple contexts or repositories.

Examples:

- identity and access boundaries
- API or event versioning rules
- data classification boundaries
- integration style guidance
- observability or audit requirements with architectural impact

## Rule Status

Every cross-cutting document should explicitly declare its status near the top:

- `Status: Mandatory`
- `Status: Recommended`

## Suggested Document Shape

- purpose
- scope
- rule
- rationale
- exceptions, if any
- related ADRs
